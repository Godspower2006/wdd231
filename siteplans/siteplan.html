Artist Collective — Site Plan

Blueprint for the three-page project: Home • Gallery & Events • Join

1. Site Name

Artist Collective — chosen because it communicates a community of local artists and a small gallery presence. Short, brandable, and suitable for both in-person exhibitions and virtual tours.

Optional domain: artistcollective.org (example)

2. Site Purpose

The site will:

  • Present current and upcoming exhibitions (dates, locations, RSVP/ticket links).
  • Show artist spotlights with portfolio thumbnails and contact/commission info.
  • Offer virtual tours, event signup forms, and membership information for artists/patrons.

3. Scenarios (target user questions)

  • Visitor: "What exhibitions are happening this weekend and how do I RSVP?"
  • Artist: "How do I submit work for the next group show and who reviews submissions?"

4. Color Schema

Defined colors (CSS variables used in styles/siteplan.css):

  • --accent #2b6cb0 — headings, primary CTAs, links.
  • --accent-2 #d4af37 — highlights, badges, membership ribbons.
  • --bg / --surface — page background and card surfaces.

5. Typography

Fonts chosen (Google Fonts):

  • Merriweather — headings, titles (serif, gallery / editorial feel).
  • Roboto — body text, nav, forms (clean, readable).

Notes

The color variables and fonts are applied across this document so the plan demonstrates the chosen schema.

6. Wireframes (Home Page)

Mobile (320–640px)

Header: Logo + Hamburger
Hero image with CTA overlay
Featured exhibition card
Artist spotlights (stacked)
Footer: Contact & Social

Desktop (≥1024px)

Hero (large image + CTA)
Featured exhibition + gallery thumbnails
Gallery grid / spotlight tiles
Upcoming Events list
Join / Signup card
Visitor info / hours

Implementation: use grid-template-areas to adapt these regions between breakpoints. Cards will be semantic (figure/figcaption, h2, address, p, button).

7. Testing & Validation

  • Run HTML validation (w3.org) and fix errors.
  • Lighthouse audits for Accessibility, Best Practices, SEO and Performance (mobile & desktop).
  • Use CSS Overview in DevTools to check color contrast (AA).
  • Optimize images to WebP where possible, set width/height, and use loading="lazy" for offscreen images.
  • Keep initial page weight under 500 KB and avoid large blocking JS/CSS.
