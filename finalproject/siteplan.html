Artist Collective — Site Plan

Blueprint for the three-page final: Home • Gallery • Join

1. Site Name

Artist Collective — chosen because it communicates a community of local artists and a small gallery presence. Short, brandable, suitable for in-person exhibitions and virtual tours.

Optional domain: artistcollective.org

2. Site Purpose

  • Present current and upcoming exhibitions (dates, locations, RSVP/tickets).
  • Show artist spotlights with portfolio thumbnails and contact/commission info.
  • Offer virtual tours, event signup forms, and membership information for artists & patrons.

3. Scenarios (target users)

  • Visitor: "What exhibitions are happening this weekend and how can I RSVP?"
  • Artist: "How do I submit work for the next group show and who reviews submissions?"

4. Color Schema

CSS variables (applied in styles/siteplan.css).

  • --accent #2b6cb0 — headings, primary CTAs, links.
  • --accent-2 #d4af37 — highlights, badges, membership ribbons.
  • --bg / --surface — page background and card surfaces.

5. Typography

  • Merriweather — headings, titles (serif editorial feel).
  • Roboto — body, nav, forms (clean, readable).

Notes

This page uses the color/typography variables so graders can preview the chosen schema.

6. Wireframes (Home Page)

Mobile (320–640px)

Header: Logo + Hamburger
Hero image with CTA
Featured exhibition
Artist spotlights (stacked)
Footer: Contact & Social

Desktop (≥1024px)

Hero (large)
Featured + thumbnails
Gallery grid
Upcoming Events list
Join / Signup card
Visitor info / hours

Implementation: use grid-template-areas for responsive regions. Cards use semantic HTML (figure/figcaption, h2, address, p, button).

7. Testing & Validation

  • Run HTML validation (w3.org) and fix errors.
  • Run Lighthouse audits: Accessibility, Best Practices, SEO (mobile & desktop).
  • Use CSS Overview for color-contrast checks (AA).
  • Optimize images to WebP, set width/height, and lazy-load offscreen images.
  • Keep initial page weight under 500 KB; avoid blocking JS/CSS.
